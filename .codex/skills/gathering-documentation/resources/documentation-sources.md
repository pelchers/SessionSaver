## Quality Assessment

### Evaluating Documentation

**Good Documentation Has**:
- Clear structure and navigation
- Up-to-date information (check dates)
- Code examples that work
- API reference
- Getting started guide
- Migration guides
- Troubleshooting section
- Regular updates

**Red Flags**:
- No version numbers
- Broken examples
- Missing API reference
- Outdated screenshots
- No recent commits/updates
- Poor organization
- No search functionality

### Version Checking

**Always check versions**:
- Framework version (React 18 vs 17)
- Language version (Python 3.10 vs 3.8)
- Package version (major.minor.patch)
- Last updated date

**Version-specific searches**:
```
"Next.js 14" documentation
"React 18" hooks guide
"Python 3.11" asyncio
```

## Best Practices

### Research Workflow

1. **Start with official docs**: Most reliable
2. **Check version**: Ensure relevance
3. **Cross-reference**: Verify across sources
4. **Test examples**: Confirm they work
5. **Note publication date**: Assess currency
6. **Save sources**: For citations

### Organization

**Folder Structure**:
```
docs/
├── official/
│   ├── nextjs/
│   ├── react/
│   └── typescript/
├── tutorials/
├── api-references/
└── examples/
```

**Naming Convention**:
```
{framework}-{topic}-{version}.md
nextjs-app-router-14.md
react-hooks-18.md
```

### Citation Format

**Markdown**:
```markdown
[Next.js Documentation](https://nextjs.org/docs) - Accessed: 2024-01-15
```

**BibTeX**:
```bibtex
@misc{nextjs_docs,
  title = {Next.js Documentation},
  url = {https://nextjs.org/docs},
  note = {Accessed: 2024-01-15}
}
```
