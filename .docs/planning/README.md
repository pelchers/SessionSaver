# SessionSaver Planning Docs

This folder contains planning artifacts for a Chrome extension concept that treats saved tab sessions like a Google Drive-style file browser.

## Files

- `PRD-Overview.md`: Product goals, audience, and scope.
- `Requirements.md`: Functional and non-functional requirements.
- `User-Flows.md`: Core interaction flows and acceptance criteria.
- `Information-Architecture.md`: Hierarchy, navigation, and metadata model.
- `Technical-Architecture.md`: Proposed extension architecture and data design.
- `Roadmap.md`: Phased delivery plan and milestones.
- `Risks-and-Open-Questions.md`: Key risks, dependencies, and unresolved decisions.
- `Wireframes.md`: Clickable wireframe spec with exact sections and component states.
- `Implementation-Backlog.md`: Epics, stories, and acceptance criteria for build execution.
- `Manifest-and-Permissions.md`: MV3 surfaces and permissions, plus minimization notes.
- `Data-Model-and-Migrations.md`: Persisted schema, versioning, and migration strategy.
- `Edge-Cases.md`: Capture/restore constraints and tricky cases (restricted URLs, etc.).
- `Library-UX-Spec.md`: Library behaviors (sort/filter/actions/empty states).
- `Session-Metadata-Spec.md`: Name/description/favorite rules and UX.
- `Explorer-Tree-Spec.md`: Tree node model, selection, keyboard, and performance notes.
- `Session-Detail-UX-Spec.md`: Detail layout, actions, responsive rules.
- `Restore-Spec.md`: Full + selective restore rules, ordering, and rate limiting.
- `Failure-Reporting-Spec.md`: Error model and user-facing messaging patterns.
- `QA-Test-Plan.md`: Manual and automated test plan with matrices.
- `Developer-Instructions.md`: Canonical local dev loop, validation gates, and pre-launch flow.
- `Release-Checklist.md`: Pre-release checklist (permissions/privacy/perf).
- `Accessibility-Checklist.md`: Keyboard, focus, semantics, contrast checklist.
- `Telemetry-and-Privacy.md`: Data handling and explicit telemetry policy for v1.
- `Threat-Model.md`: Threats, mitigations, and security posture for a local-only extension.
- `Store-Listing-Draft.md`: Draft Chrome Web Store listing copy and disclosures.
- `Settings-Spec.md`: Settings and management features (clear all data, import/export roadmap).
- `Search-Spec.md`: Search scope, ranking, and UI behavior (post-MVP or optional).
- `adr/README.md`: ADR-style phased preplanning and phase artifacts.
